Three Approaches for the Basics Section

Same 4 pages, different philosophies for how users experience them

A

"Mission Control Academy"

Structure: Each page is a mission briefing. Short intro text sets the scene, then a sequence of standalone interactive widgets teach each concept. Scroll down = next experiment.

Interactives: Self-contained simulations per concept. Circuit builder, battery cell cross-section, grid balancing game, storage tech comparison tool. Each widget is a "station" you can play with independently.

Vibe: Science museum meets spy HQ. Each widget feels like a classified experiment you're running. Glowing borders, data readouts, status indicators.

Effort: ~15-20 interactive components across 4 pages. Each is modular and independent.

Pros: Modular (can build/ship incrementally), each interactive is reusable, easy to maintain

Cons: Less narrative cohesion between widgets, potentially feels like a textbook with fancy illustrations

B

"Explorable Explanations"

Structure: Each page is one continuous scroll-driven experience. As you scroll, the visualization transforms and responds. Text and interactive are fused — you can't separate them.

Interactives: The entire page IS the interactive. Scroll position drives animation state. Sticky canvas elements that morph as you read. Inline sliders/toggles that modify the ongoing visualization in real-time.

Vibe: Like a Bartosz Ciechanowski deep-dive or an NYT visual essay, but in mission-control dark mode. Immersive and cinematic.

Effort: ~4 large scroll-driven experiences. Each is a single complex component managing many states.

Pros: Most immersive and memorable, strongest narrative flow, truly unique

Cons: Highest complexity per page, harder to maintain, scroll-driven state machines are tricky, mobile experience needs extra work

C

"Hybrid: Briefings + Labs"

Structure: Each page has 2 layers. A "briefing" layer (prose + embedded scroll-driven animations for key concepts) and "lab" sections (standalone interactive playgrounds you can explore freely). Best of both worlds.

Interactives: 2-3 scroll-driven explorable sections per page for the core narrative, plus 1-2 standalone "lab" simulations at the end or inline for deeper exploration.

Vibe: Read the briefing, then run the experiment. The briefing builds intuition, the lab lets you test it.

Effort: ~8-10 scroll-driven sections + ~8-10 standalone labs across 4 pages. Medium-high complexity.

Pros: Narrative cohesion from scroll-driven parts, replayability from standalone labs, can ship briefings first and add labs iteratively

Cons: Two interaction paradigms on one page could feel inconsistent if not designed carefully

My Recommendation: C (Hybrid)

The explorable explanation approach (B) creates the most memorable experience, but it's fragile and hard to iterate on. The modular approach (A) is practical but risks feeling like a widget catalog. The hybrid (C) gives you the cinematic scroll-driven storytelling where it matters most (the "aha moment" concepts) while also offering standalone labs that are easier to build, test, and maintain. You can ship the briefing layer first and progressively add labs.
